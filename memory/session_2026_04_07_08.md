## What Was Built (April 7-8, 2026)

### Core Architecture — WORKING
- Plugin loads via RE_Kenshi mod loader with `startPlugin()` export (C++ mangled name)
- Game loop hooked via `KenshiLib::GetRealAddress` + `AddHook` using `GetProcAddress("KenshiLib.dll", mangled_name)` to find the stub
- All game code runs on the game thread (no threading issues)
- Compiled with VS2010 v100 x64 toolset for KenshiLib ABI compatibility
- ENet also compiled with v100 (fixed C99 for-loop declarations in protocol.c)

### Networking — WORKING
- ENet UDP client connects to dedicated server
- F9 hotkey: quick connect to localhost:7777
- Auto-reconnect every 3 seconds when connection drops
- Server relays player state between clients
- Player position read from `ch->getPosition()` and sent at 20Hz
- `enet_host_flush()` after every send (prevents server timeout)
- Client timeout increased to 60 seconds (handles alt-tab)

### Host NPC Sync (Phase 1) — PARTIALLY WORKING
- Host client scans `ou->getCharacterUpdateList()` for nearby NPCs
- Proximity filter: 500 unit radius from any player character
- Sequential NPC ID system (Character* pointer → uint32_t)
- NPC_SPAWN_REMOTE, NPC_BATCH_STATE (10Hz), NPC_DESPAWN_REMOTE packets
- Server tracks host player (`is_host` flag), relays NPC packets to joiners only
- **Not tested with a second client yet** — need two Kenshi instances

### MyGUI UI — WORKING
- F8: Connect dialog (host/port input, Connect/Disconnect buttons)
- Status text top-left showing connection state
- Chat window (read-only display + input + Send button)
- F10: Admin panel (host only)
- **Must use base skins** (`Kenshi_TextBoxEmptySkin`, `Kenshi_EditBoxEmptySkin`, `Kenshi_Button1Skin`) with explicit `setFontName` and `setTextColour`
- Layout template skins (`Kenshi_TextboxStandardText` etc.) are `ResourceLayout`, NOT widget skins — they don't apply font/color when used directly

### Admin Panel (F10) — WORKING
- Connected players list with positions (updates every 0.5s)
- Synced NPC count
- "Spawn Test NPC Here" button — spawns NPC at player position
- "Clear Remote NPCs" button (not yet implemented)
- Give Items: 6 buttons (Katana, Plank, Crossbow, Armour, Medkit, Food)

### Item Giving — PARTIAL
- **Working:** Dried Meat, Standard first aid kit, Armoured Rags (gives blueprint though)
- **Failing:** Katana, Plank (createItem returns NULL — weapons need weapon mesh GameData)
- **Not found:** Toothpick (crossbow name is different — bolts are "Bolts [Toothpicks]")

## What Failed and Why

### VS2022 (v143) toolset
- `KenshiLib::GetRealAddress` assertion fails — member function pointer ABI mismatch
- Background thread approach crashes — KenshiLib objects not thread-safe
- Ogre::FrameListener only fires once during splash screen, not during gameplay
- Boost 1.60 `std::unary_function` removed in C++17
- `s_host` variable name conflicts with winsock macro

### VS2010 (v100) toolset requirements
- No `enum class` → use namespace with `static const uint8_t`
- No `constexpr` → use `static const`
- No `nullptr` → use `NULL`
- No `auto` → use explicit iterator types
- No `std::function` → use typedef function pointer
- No `<mutex>`, `<queue>` — not in v100 STL
- `std::to_string(uint32_t)` ambiguous → use `std::ostringstream`
- `#include <kenshi/Item.h>` causes static init crash — use forward declaration from Character.h instead

### NPC Spawning
- `createRandomCharacter(NULL, ...)` crashes — needs valid Faction pointer
- `npc->ai = NULL` crashes on next frame — game update loop references AI
- Use `ou->factionMgr->getEmptyFaction()` for neutral NPCs
- Use `ou->player->getFaction()` for friendly NPCs (but they join your team)

### RE_Kenshi Setup
- Manual install missed `RE_Kenshi.dll` (was in install/ subfolder, not obvious)
- `IgnoreHash: true` alone doesn't fix version mismatch — need the installer to downgrade exe
- Plugin needs `RE_Kenshi.json` file with `"Plugins": ["KenshiMP.dll"]`
- Plugin needs `KenshiMP.mod` binary file (32-byte stub: type=16, version=1, all zeros)
- Mod must be enabled in Kenshi's mod list

### GetRealAddress / AddHook
- `_NV_mainLoop_GPUSensitiveStuff` is a **zero-byte empty stub** — can't use with GetRealAddress
- Virtual member function pointers contain vtable indices, not addresses
- Solution: `GetProcAddress(GetModuleHandleA("KenshiLib.dll"), "?mainLoop_GPUSensitiveStuff@GameWorld@@UEAAXM@Z")` → gets the actual stub address in KenshiLib.dll's FUNC_BEGIN/FUNC_END range

## Build Commands

### Core DLL (v100):
```bash
cmake -B build -T v100 -A x64 \
  -DENET_DIR=deps/enet2 \
  -DKENSHILIB_DIR=deps/KenshiLib \
  -DKENSHILIB_EXAMPLES_DEPS=deps/KenshiLib_Examples_deps \
  -DBOOST_ROOT=deps/KenshiLib_Examples_deps/boost_1_60_0 \
  -DKENSHIMP_BUILD_CORE=ON -DKENSHIMP_BUILD_SERVER=OFF -DKENSHIMP_BUILD_INJECTOR=OFF
cmake --build build --config Release --target KenshiMP
```

### Server (VS2022):
```bash
cmake -B build_server -A x64 -DENET_DIR=deps/enet2 \
  -DKENSHIMP_BUILD_CORE=OFF -DKENSHIMP_BUILD_SERVER=ON -DKENSHIMP_BUILD_INJECTOR=OFF
cmake --build build_server --config Release
```

### Deploy:
```bash
cp build/core/Release/KenshiMP.dll "/c/Program Files (x86)/Steam/steamapps/common/Kenshi/mods/KenshiMP/"
```
