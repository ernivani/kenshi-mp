## What Was Built (April 8, 2026)

### Phase 1 NPC Sync — VERIFIED WORKING
- Host scans `ou->getCharacterUpdateList()` every tick
- Proximity filter: 500 unit radius from any player
- NPC ID system: Character* pointer → sequential uint32_t
- NPC_SPAWN_REMOTE (reliable), NPC_BATCH_STATE (20Hz unreliable), NPC_DESPAWN_REMOTE (reliable)
- Server tracks host (`is_host` flag), relays NPC packets to joiners only
- Joiner spawns remote NPCs with `createRandomCharacter` using player faction
- Immediate teleport on batch state update (fights game AI)
- Host resends all synced NPCs when a new joiner connects
- Host skips player avatars in NPC scan (prevents joiner seeing themselves duplicated)
- **Tested end-to-end with two real Kenshi instances**

### Test Client Tool
- `tools/test_client.cpp` — console app that connects as non-host joiner
- Logs all received packets (NPC spawn, batch state, despawn, player state, chat)
- Verified the full NPC sync pipeline before testing with real second instance

## What Failed and Why

### createRandomCharacter failures
- `getEmptyFaction()` as faction → returns NULL on some instances. **Use `ou->player->getFaction()` instead**
- Race GameData as charTemplate → returns NULL. Race data is wrong type for character template. **Pass NULL for template**
- RaceData from `RaceData::getRaceData(stringID)` resolves but passing its `data` as charTemplate fails

### NPC hiding on joiner
- Teleporting NPCs underground (-99999 y) works initially
- Continuous hide loop was hiding our own synced NPCs (fixed with std::set lookup)
- Game continuously spawns new NPCs that need re-hiding
- **Currently disabled** — needs better approach (maybe destroy instead of teleport)

### NPC sync timing
- Host starts scanning before joiner connects → joiner misses initial spawn packets
- **Fixed**: host resends all synced NPCs when new player joins (SPAWN_NPC triggers resend)

### Host scanning player avatars
- Host's NPC scan included remote player avatars spawned by npc_manager
- Joiner received their own player back as a synced NPC
- **Fixed**: `npc_manager_is_player_npc()` check in host_sync_tick

### GameData field names
- `GameData::stringID` (offset 0x58) = composite key like "576-gamedata.base.TECH.1"
- `GameData::name` (offset 0x28) = display name like "Katana"
- `gamedataSID` map keyed by composite stringID, NOT by display name
- Must iterate map and compare `.name` field for item lookup

### Item creation
- `createItem(GameData*)` simple overload returns NULL for everything
- `createItem(gd, hand(), NULL, NULL, -1, NULL)` full overload works for food/armor
- Weapons (Katana, Plank) still return NULL — need weapon mesh GameData parameter
- `#include <kenshi/Item.h>` causes static init crash — use forward declaration from Character.h

## Known Issues (Not Fixed Yet)
- Synced NPCs join joiner's team (using player faction because getEmptyFaction fails)
- Local NPC hiding disabled (continuous hide was interfering with synced NPCs)
- No animations on synced NPCs (animation_id always 0)
- NPCs are random appearance (race template not applied — createRandomCharacter ignores it)
- Player yaw always 0 (rotation not synced)

## Architecture Summary

```
Host Kenshi → host_sync.cpp scans NPCs → ENet → Server relays → ENet → Joiner Kenshi
                                                                         ↓
                                                                   npc_manager.cpp
                                                                   spawns + teleports
                                                                   remote NPCs
```

  -DENET_DIR=deps/enet2 \
